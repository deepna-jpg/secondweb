단순한 설명을 넘어, **"왜 이렇게 짰는지"**와 **"어떤 원리인지"**를 아주 자세하게 풀어 쓴 심화 노트입니다. 코드를 보다가 이해가 안 갈 때 교과서처럼 읽어보세요! 🧐

## 📂 1. 프로젝트 폴더 구조와 파일 역할 (심화)

이 프로젝트는 `Vite` + `React`로 만들어졌으며, **파일을 역할별로 폴더에 나눠 담는 방식**을 따르고 있습니다.

### ⚙️ 시스템 설정 및 진입 (System)
**웹사이트가 켜지고 작동하는 기본 뼈대**입니다.

#### 1. `src/main.tsx` (전원 스위치 & 입장문)
> **비유**: 이삿짐 센터가 가구(리액트 앱)를 빈 집(`index.html`)에 배치하는 작업입니다.
- **상세 역할**: 웹 브라우저는 리액트를 모릅니다. 오직 HTML만 알죠. 이 파일은 실제 HTML 파일(`index.html`) 속에 있는 `<div id="root"></div>`라는 빈 상자를 찾아내서, 우리가 만든 리액트 앱 전체(`App`)를 그 안에 쏙 집어넣고 실행시키는 역할을 합니다.
- **핵심 코드**: `createRoot(...)` (뿌리 내리기), `render(...)` (화면에 그리기)

#### 2. `src/App.tsx` (지도 & 교통정리)
> **비유**: 건물 1층의 안내 데스크입니다. "201호로 가세요", "지하 1층으로 가세요"라고 안내합니다.
- **상세 역할**: **라우팅(Routing)**을 담당합니다. 사용자가 주소창에 `/`를 치면 홈을 보여주고, `/team`을 치면 팀 소개를 보여주라고 규칙을 정해둔 곳입니다.
- **중첩 라우팅 (Nested Routing)**:
  - `Layout`이라는 큰 틀 안에 `HomePage`, `TeamPage` 같은 작은 화면들을 자식으로 넣었습니다.
  - 이렇게 하면 주소가 바뀌어도 `Layout`(헤더, 푸터)은 그대로 있고, 그 안의 내용물만 샥샥 바뀝니다.

#### 3. `src/index.css` (전역 스타일 옷장)
> **비유**: 전 직원이 입어야 하는 유니폼 규칙입니다.
- **상세 역할**: 모든 파일에서 공통으로 쓸 디자인 규칙을 모아둡니다.
- **CSS 변수(`:root`)**: `--primary-color: #fff4b0;` 처럼 색상에 이름을 붙여둡니다. 나중에 색을 바꿀 때 여기서 한 번만 바꾸면 웹사이트 전체 색상이 한방에 바뀝니다. (유지보수의 핵심!)

#### 4. `.env` (비밀번호 금고)
> **비유**: 은행의 개인 대여 금고입니다.
- **상세 역할**: Gemini API 키처럼 외부에 노출되면 안 되는 **비밀 정보**를 저장하는 파일입니다. 
- **보안**: 이 파일은 GitHub 같은 곳에 절대 올리지 않습니다. (`.gitignore`에 등록되어 보호받습니다.)
- **사용법**: `VITE_`라는 이름으로 시작해야 하며, 코드에서는 `import.meta.env.VITE_...`로 가져다 씁니다.

### 🧱 컴포넌트 폴더 (`src/components/`)
**여러 페이지에서 `재사용`하는 조립 부품**들을 모아두는 곳입니다.

#### 4. `Layout.tsx` (마스터 템플릿 / 액자)
> **비유**: 그림은 바뀌어도 액자는 그대로죠? 페이지의 '액자'입니다.
- **왜 따로 만들었나요?**: 모든 페이지마다 헤더와 푸터를 복사-붙여넣기 하면 코드가 너무 길어지고 수정이 힘듭니다. 그래서 틀만 따로 만든 것입니다.
- **핵심 기술**:
    1.  **Sticky Footer**: 내용이 아무리 적어도 푸터(바닥글)가 화면 중간에 떠 있지 않고 맨 아래에 붙어있게 하는 기법 (`min-height: 100vh` + `flex-direction: column` + `flex-grow: 1`).
    2.  **Outlet**: 리액트 라우터에서 제공하는 **'다목적 구멍(Slot)'**입니다. "여기엔 페이지 내용이 들어올 거야"라고 자리를 비워두는 것입니다.

- **Active Style**: `isActive`라는 정보를 받아서 "지금 버튼이 눌려있다"는 걸 감지합니다. 이걸로 배경색을 노랗게 칠해주는 것입니다.

#### 6. `WeatherWidget.tsx` (날씨 & AI 코디네이터)
> **비유**: 매장 입구에 있는 '오늘의 추천 코디' 마네킹입니다.
- **상세 역할**: 현재 서울의 기온을 보여줄 뿐만 아니라, **Gemini AI**를 통해 이 날씨에 어떤 옷을 입으면 좋을지 친절하게 알려줍니다.
- **UX 포인트**: AI가 답변을 생각하는 동안 "고민 중... 🤔"이라는 메시지를 띄워 사용자에게 현재 진행 상황을 알려주는 친절한 인터페이스를 갖추고 있습니다.

### 📄 페이지 폴더 (`src/pages/`)
**실제 사용자가 보게 될 `화면(Screen)`** 하나하나입니다.

#### 6. `HomePage.tsx` (메인 화면)
> **비유**: 가게의 쇼윈도입니다. 가장 예쁘고 중요한 것만 보여줍니다.
- **구조**:
    1.  **Hero Section**: '미래형 웹에 오신 것을 환영합니다'라는 인사말과 함께 최신 웹 트렌드를 소개합니다.
    2.  **Grid Layout**: `display: grid`를 사용해 프리미엄 디자인, 압도적인 속도, 완벽한 반응형이라는 3가지 핵심 가치를 카드 형태로 보여줍니다.
- **학습 포인트**: `className="clean-card"`처럼 미리 만들어둔 디자인(CSS 클래스)을 가져와서 입히는 방법을 배울 수 있습니다.

#### 7. `TeamPage.tsx` (데이터 목록 화면)
> **비유**: 엑셀 명단을 보기 좋게 명찰로 뽑아내는 작업입니다.
- **데이터 배열 (`teamMembers`)**:
    - 변하지 않는 정보(이름, 역할)를 `const` 변수에 배열(`[]`) 형태로 저장했습니다.
    - 실제 실무에서는 이 데이터를 서버(백엔드)에서 받아오지만, 지금은 연습이라 직접 적어두었습니다(하드코딩).
- **`map` 함수 (반복문)**:
    - 리액트에서 가장 중요한 기술 중 하나입니다.
    - "데이터가 4개니까 카드도 4장 찍어내라"는 명령입니다. 코드를 4번 복사할 필요 없이 한 번만 짜면 됩니다.
- **`key` 속성**:
    - 리액트가 카드를 관리할 때 쓰는 **주민등록번호**입니다. 반복문으로 요소를 찍어낼 땐 반드시 고유한 번호(`id`)를 `key`로 줘야 성능 문제가 생기지 않습니다.

## 📚 2. 핵심 용어 심화 설명

### `const` (상수)
- **뜻**: Constant, '변하지 않는 수'
- **설명**: 한 번 이름을 붙이면(선언하면) 그 안의 내용을 다른 걸로 갈아치울 수 없습니다.
- **왜 쓰나요?**: 실수로 데이터를 덮어쓰는 사고를 막기 위해 "이건 고정이야!"라고 못 박아두는 것입니다. (리액트 컴포넌트나 함수도 값이 변하면 안 되니 보통 `const`로 만듭니다.)

### `style={{ ... }}` (인라인 스타일)
- **왜 중괄호가 2개인가요?**: 
    1. 바깥쪽 `{ }`: "여기서부터는 HTML이 아니라 자바스크립트 문법을 쓸 거야!"
    2. 안쪽 `{ }`: "이건 스타일 정보를 담은 **객체(Object)** 덩어리야!"
- **특징**: CSS 파일로 왔다 갔다 할 필요 없이 바로 디자인을 입힐 수 있어 편하지만, 코드가 지저분해질 수 있어 보통은 `className`을 더 권장합니다. 저희는 학습용이라 둘 다 써봤습니다.

### `import` / `export`
- **import (수입)**: 다른 파일에 있는 부품을 가져오는 것.
- **export (수출)**: 내가 만든 부품을 다른 파일이 가져갈 수 있게 내놓는 것.

### `useState` (상태 만들기)
- **뜻**: "화면의 상태(State)를 쓰겠다(Use)"는 선언
- **설명**: 리액트에게 "이 변수는 화면에 보일 거니까, 값이 바뀌면 즉시 화면을 다시 그려줘!"라고 부탁하는 도구입니다. 리액트 개발의 80%를 차지하는 가장 중요한 기능입니다.

### `async` / `await` (비동기 처리)
- **뜻**: 비동기(Asynchronous)와 기다림(Await)
- **설명**: 인터넷에서 데이터를 가져오는 건 시간이 좀 걸립니다. 이때 코드가 멈추지 않고 "데이터 가져올 때까지 잠깐 기다려줘, 그동안 다른 일 하고 있을게!"라고 말하는 똑똑한 약속 방식입니다.

## 🎁 [부록] 파일 확장자 대백과 (File Extensions)

파일 이름 끝의 꼬리표(`.ts`, `.tsx` 등)가 무엇을 의미하는지 정리했습니다.

| 확장자 | 성격 | 비유 | 주요 역할 |
| :--- | :--- | :--- | :--- |
| **.js** | 자바스크립트 | 생머리(기본) | 웹의 동작을 담당하는 가장 기초적인 언어입니다. |
| **.ts** | 타입스크립트 | 안전모 쓴 두뇌 | JS에 '안전 장치'를 추가한 것입니다. 실수를 미리 잡아줍니다. |
| **.tsx** | 리액트 화면 | 종합 선물 세트 | 리액트 컴포넌트를 만들 때 씁니다. (디자인 + 기능) |
| **.css** | 스타일시트 | 예쁜 옷/메이크업 | 웹사이트를 예쁘게 꾸미는 디자인 설계도입니다. |
| **.json** | 데이터/설정 | 장부/목록표 | 프로그램 코드가 아니라, 정보(값)만 담고 있는 서류 가방입니다. |
| **.md** | 마크다운 | 깔끔한 메모장 | 지금 읽고 계신 이 파일처럼, 서식이 있는 문서를 작성할 때 씁니다. |
| **.env** | 환경 변수 | 비밀 금고 | API 키 같은 보안 정보를 숨겨두는 특수 파일입니다. |
